## 2. рагмент коду (компонент Input)
``jsx
// src/components/atoms/Input/Input.jsx
import styles from './Input.module.css';

const Input = ({ type = 'text', placeholder, label }) => {
  return (
    <div className={styles.container}>
      {label && <label className={styles.label}>{label}</label>}
      <input type={type} placeholder={placeholder} className={styles.input} />
    </div>
  );
};

export default Input;
``

## 3. ідповіді на контрольні питання
1. **Що таке Atomic Design і навіщо ми розділяємо компоненти на атоми та молекули?**  
   Atomic Design — це методологія створення дизайн-систем, яка передбачає розбиття інтерфейсу на дрібні, незалежні компоненти, починаючи від найпростіших (атомів) до складніших (молекул, організмів і сторінок). озділення на атоми та молекули дозволяє підвищити перевикористання коду, спростити тестування та зробити архітектуру проєкту більш логічною й масштабованою.

2. **Як працюють CSS Modules і як вони вирішують проблему глобальних імен класів?**  
   CSS Modules генерують унікальні імена класів під час збірки проєкту (наприклад, Button_primary__2s1d3). е гарантує, що стилі, написані для одного компонента, автоматично ізолюються і не вплинуть на інші елементи сторінки. Таким чином вирішується проблема глобальних конфліктів імен класів у звичайному CSS.

3. **Що таке props.children і в якому компоненті цієї лабораторної роботи ми його використали?**  
   props.children — це спеціальна властивість (проп) в React, яка містить вкладений контент, переданий компоненту між його відкриваючим і закриваючим тегами.  цій лабораторній роботі ми застосовували його в компонентах Button та Card, щоб вони могли відображати будь-який вміст (текст або інші компоненти), переданий всередину.

4. **ому атрибут HTML class у JSX записується як className?**  
   скільки JSX компілюється в JavaScript, а в JavaScript слово class є зарезервованим ключовим словом (для створення класів), розробники React ввели атрибут className як аналог HTML-атрибуту class, щоб уникнути синтаксичних конфліктів.
