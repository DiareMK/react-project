# Відповіді на контрольні запитання (Практична робота №4)

## 1. У чому головна різниця між керованими та некерованими компонентами?

Основна різниця полягає в тому, **де зберігаються дані форми**.
- **Керовані компоненти (Controlled):** Дані зберігаються в стані React (`useState`). Кожен ввід користувача оновлює стан, а значення інпуту (`value`) синхронізується зі станом. Це дозволяє React повністю контролювати компонент.
- **Некеровані компоненти (Uncontrolled):** Дані зберігаються безпосередньо в DOM. Для отримання значень використовуються `ref` (хук `useRef`). Стан React не оновлюється при кожному натисканні клавіші.

## 2. Навіщо використовувати useRef при роботі з формами і коли це виправдано?

`useRef` використовується для створення посилання на DOM-елемент. Це виправдано у таких випадках:
- Створення **некерованих компонентів** для простих форм, де не потрібна миттєва валідація при вводі, щоб уникнути зайвих перемальовувань (re-renders).
- Встановлення **фокусу** на поле вводу програмно (наприклад, після відкриття модального вікна).
- Інтеграція зі сторонніми бібліотеками, які працюють безпосередньо з DOM.
- Читання значень файлів (`<input type="file" />`), оскільки вони завжди є некерованими в React.

## 3. Чому важливо викликати event.preventDefault() при обробці події submit?

Стандартна поведінка браузера при відправці форми (`submit`) — це перезавантаження поточної сторінки або перехід за вказаним у `action` URL. 
В Single Page Applications (SPA), якими є React-додатки, ми хочемо обробляти дані самостійно через JavaScript без перезавантаження сторінки. `event.preventDefault()` зупиняє цю стандартну дію браузера, дозволяючи нам виконати валідацію, оновити стан або відправити запит до API "фоново".
