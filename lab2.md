## 3. Що станеться, якщо видалити атрибут key?
Якщо видалити атрибут key під час рендерингу списку за допомогою .map(), React видасть попередження в консолі розробника: Warning: Each child in a list should have a unique "key" prop. а практиці це означає, що при будь-якій зміні списку (додаванні, видаленні або переміщенні елементів) React не зможе ефективно відстежити, які саме елементи змінилися. амість того, щоб точково оновити DOM-вузли (ідентифікуючи їх за ключем), React змушений буде перемальовувати або перестворювати більше елементів з нуля, що призведе до значного падіння продуктивності, а в інтерактивних компонентах (наприклад, з інпутами) локальний стан (state) може прив'язатися до неправильного елемента, викликаючи баги відображення.

## 4. ідповіді на контрольні запитання
1. **авіщо потрібен проп key у списках React?**  
   роп key діє як унікальний ідентифікатор для кожного елемента списку в React. ін дозволяє алгоритму Reconciliation швидко і точно визначати, які елементи були додані, змінені чи видалені, що є критично важливим для оптимізації оновлень DOM і збереження локального стану складених компонентів.

2. **ому не рекомендується використовувати індекс масиву (index) як ключ?**  
   Якщо порядок елементів у масиві зміниться (наприклад, сортування, додавання нового елемента на початок масиву, або видалення), їхні індекси також зміняться. React, дивлячись на ключі (які були індексами), вирішить, що самі елементи змінились, хоча вони просто змінили позицію. е не тільки негативно впливає на продуктивність через зайве перемалювання, але й може викликати збій у збереженні стану (state) компонентів (наприклад, введений текст у полі введення "залишиться" на старому індексі).

3. ** чому різниця між імперативним створенням списку (цикл for) та декларативним (map)?**  
   мперативний підхід (через or) вимагає ручного керування процесом: ми маємо створити порожній масив, описати крок за кроком як пройтись по даним, і самостійно додати (push) туди JSX-елементи. екларативний підхід (наприклад, .map()) фокусується на тому, **що** ми хочемо отримати. и просто вказуємо перетворення кожного елемента даних у JSX, а метод .map() сам турбується про створення і повернення нового масиву елементів. од стає чистішим і коротшим.

4. **Як передати всі властивості об'єкта в компонент одним рядком?**  
   ожна використати Spread operator (оператор розширення): {...item}. априклад: <Post key={item.id} {...item} />. е автоматично передасть всі ключі об'єкта item як окремі пропси компонента Post.
